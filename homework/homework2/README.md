# HSEBank Finance System

Полнофункциональная система управления финансами с CLI-интерфейсом, реализующая лучшие практики ООП и паттерны
проектирования.

## Функциональность

### Основные модули:

- **Управление счетами** - создание, просмотр, обновление, удаление банковских счетов
- **Управление категориями** - категоризация доходов и расходов
- **Управление операциями** - проведение финансовых операций с автоматическим пересчетом балансов
- **Аналитика и отчеты** - анализ финансовой деятельности, группировка по категориям
- **Импорт/экспорт данных** - поддержка JSON, CSV, YAML форматов
- **Системные инструменты** - мониторинг производительности, управление кэшем

### Дополнительные возможности:

- **Кэширование данных** через паттерн Proxy
- **Валидация ввода** - надежная обработка пользовательских данных
- **Логирование операций** - детальное отслеживание выполнения
- **Демонстрация паттернов** - учебные примеры реализации

## Архитектура и принципы

### SOLID принципы:

| Принцип | Реализация                              | Примеры классов                                           |
|---------|-----------------------------------------|-----------------------------------------------------------|
| **SRP** | Каждый класс имеет одну ответственность | `AccountManagementService`, `CategoryManager`             |
| **OCP** | Легкое расширение функциональности      | `DataImporter`, `ICommand` иерархия                       |
| **LSP** | Взаимозаменяемость наследников          | `RepositoryProxy`, команды, импортеры                     |
| **ISP** | Специализированные интерфейсы           | `IFinancialCommand`, `StructuredFinancialVisitor`         |
| **DIP** | Зависимость от абстракций               | Все сервисы используют `IRepository`, `IFinancialFactory` |

### GRASP принципы:

- **Information Expert** - логика в наиболее подходящих классах (`BalanceService`, `AnalyticsFacade`)
- **Creator** - централизованное создание объектов (`FinancialFactory`)
- **Controller** - координация работы системы (`FinanceCLI`, менеджеры)
- **Low Coupling** - слабая связность через интерфейсы и DI
- **High Cohesion** - четкое разделение ответственности

## Паттерны проектирования

### Порождающие:

| Паттерн            | Назначение                         | Классы                                  |
|--------------------|------------------------------------|-----------------------------------------|
| **Factory Method** | Централизованное создание объектов | `FinancialFactory`, `IFinancialFactory` |
| **Singleton**      | Единственный экземпляр сервисов    | Spring `@Service`, `@Component`         |

### Структурные:

| Паттерн       | Назначение                               | Классы                                 |
|---------------|------------------------------------------|----------------------------------------|
| **Proxy**     | Прозрачное кэширование                   | `RepositoryProxy`, `RepositoryConfig`  |
| **Decorator** | Динамическое добавление функциональности | `TimingDecorator`                      |
| **Facade**    | Упрощение сложных подсистем              | `AnalyticsFacade`, `BankAccountFacade` |

### Поведенческие:

| Паттерн             | Назначение                            | Классы                                      |
|---------------------|---------------------------------------|---------------------------------------------|
| **Command**         | Инкапсуляция запросов, поддержка undo | `ICommand`, `BaseCommand`, команды операций |
| **Template Method** | Алгоритмы с переопределяемыми шагами  | `DataImporter` и наследники                 |
| **Strategy**        | Взаимозаменяемые алгоритмы            | `ExportService` с visitorStrategies         |
| **Visitor**         | Операции над разнородными объектами   | `FinancialVisitor`, экспортеры              |
