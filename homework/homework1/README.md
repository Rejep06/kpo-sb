# Система управления Московским зоопарком

## Описание проекта

Данное приложение представляет собой ERP-систему для автоматизации учета животных и инвентаря Московского зоопарка.
Система позволяет вести учет животных, проверять их здоровье перед приемкой, рассчитывать общее потребление пищи и
формировать списки животных для контактного зоопарка.

## Архитектура и принципы SOLID

1. SRP (Single Responsibility Principle)
   **Каждый класс имеет одну четкую ответственность:**
    * InventoryFactory - создание объектов инвентаря
    * MoscowZooService - бизнес-логика зоопарка
    * MoscowVetClinic - проверка здоровья животных
    * Console - взаимодействие с пользователем

2. OCP (Open-Closed Principle)
    * Система легко расширяема для добавления новых типов животных и вещей
    * Интерфейсы Inventory, Alive, Herbivore позволяют добавлять новые сущности без изменения существующего кода

3. LSP (Liskov Substitution Principle)
    * Все подклассы животных (Rabbit, Monkey, Tiger, Wolf) могут использоваться вместо базовых классов Animal,
      HerbivoreAnimal, PredatorAnimal
    * Вещи (Table, Computer) корректно заменяют базовый класс Thing

4. ISP (Interface Segregation Principle)
   **Интерфейсы разделены по функциональности:**
    * Inventory - для инвентаризации
    * Alive - для живых существ
    * Herbivore - для травоядных животных
    * VetClinic - для проверки здоровья

5. DIP (Dependency Inversion Principle)
    * Console зависит от интерфейса ZooService, а не от конкретной реализации
    * Можно создать другую реализацию ZooService (например, PremiumZooService) без изменения кода Console
    * MoscowZooService (высокоуровневый модуль) не знает о конкретных реализациях VetClinic
    * Можно легко заменить MoscowVetClinic на другую реализацию (например, PrivateVetClinic) без изменения кода
      MoscowZooService

## Структура проекта

### Доменная модель

#### Интерфейсы:

    Inventory - инвентаризируемые объекты (номер и название)
    Alive - живые существа (потребление пищи)
    Herbivore - травоядные животные (уровень доброты)

#### Классы животных:

    Animal - абстрактный базовый класс
    HerbivoreAnimal - травоядные животные
    PredatorAnimal - хищные животные
    Rabbit, Monkey, Tiger, Wolf - конкретные реализации

#### Классы вещей:

    Thing - абстрактный базовый класс
    Table, Computer - конкретные реализации

### Сервисный слой

    MoscowZooService - основной сервис с бизнес-логикой
    InventoryFactory - фабрика для создания объектов
    MoscowVetClinic - сервис проверки здоровья животных
    Console - консольный интерфейс пользователя

## Функциональные возможности

### Основные функции:

1. Добавление животных - с проверкой здоровья ветеринарной клиникой
2. Добавление вещей - в инвентарь зоопарка
3. Расчет общего потребления пищи - суммарно всеми животными
4. Формирование списка для контактного зоопарка - животные с уровнем доброты > 5
5. Генерация отчетов - полная инвентаризация зоопарка

### Меню приложения:

1. Добавить объект в инвентарь
2. Показать общее потребление пищи
3. Показать животных для контактного зоопарка
4. Сгенерировать отчет по инвентарю
5. Показать всех животных
6. Показать все объекты инвентаря
7. Выход
